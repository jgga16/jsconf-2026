# Decision: Talk & Asset Data Schema

**Author:** Mikey (Lead)  
**Date:** 2026-03-07  
**Status:** proposed  
**Affects:** Brand, Andy, Data

## Context

Every agent touches the data shape: Brand produces it, Andy consumes it, Data reads it to generate images. We need one contract.

## Decision

### File location

```
src/data/talks.json       ← committed to git (source of truth)
src/data/assets.json      ← generated by image pipeline, also committed
```

### Talk schema (`talks.json`)

```jsonc
{
  "event": {
    "name": "JSConf España 2026",
    "date": "2026-03-14",
    "venue": "La Nave, Madrid"
  },
  "talks": [
    {
      "id": "retro-javascript",            // kebab-case slug from title
      "title": "Retro JavaScript: rehaciendo el pasado con código moderno",
      "description": "Aprende las últimas novedades...",
      "speakers": [                         // array — supports multi-speaker talks
        {
          "name": "Carmen Ansio",
          "role": "Designer Engineer @Stripe",
          "avatarUrl": "https://www.jsconf.es/img/carmen-ansio.webp"
        }
      ],
      "time": {
        "start": "09:40",
        "end": "10:20"
      },
      "type": "talk",                       // "talk" | "break" | "ceremony" | "lightning"
      "language": "es",                     // "es" | "en"
      "tags": []                            // optional, for future filtering
    }
  ]
}
```

### Asset schema (`assets.json`)

```jsonc
{
  "assets": [
    {
      "talkId": "retro-javascript",         // FK to talks[].id
      "cover": "/covers/retro-javascript.png",        // cartridge cover (template-composed)
      "avatar8bit": "/avatars/carmen-ansio-8bit.png",  // pixelated avatar
      "avatarOriginal": "/avatars/carmen-ansio.webp"   // downloaded original
    }
  ]
}
```

### Generated files location

```
public/
  covers/          ← cartridge cover PNGs (256×384 or similar 2:3 ratio)
  avatars/         ← 8-bit avatar PNGs (128×128, nearest-neighbor upscaled)
```

### Validation

Brand must provide a Zod schema (`src/schemas/talk.ts`) that validates `talks.json` at build time. Build fails if data is invalid.

## Key rules

1. `id` is the join key between talks and assets — always kebab-case, derived from title
2. `speakers` is always an array (never a single object) — future-proofs multi-speaker talks
3. `type` field lets Andy filter out breaks/ceremonies from cartridge rendering
4. Asset paths are relative to `public/` for Astro's static asset handling
5. Both JSON files are committed to git — they're the cached source of truth

## Consequences

- Brand and Data can work in parallel once this schema is agreed
- Andy can start with mock data immediately
- Zod validation catches drift early
